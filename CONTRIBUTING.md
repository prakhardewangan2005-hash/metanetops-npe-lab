# Contributing Guidelines

Thank you for your interest in contributing to this Network Operations lab.
Contributions are welcome to improve reliability practices, documentation,
and operational workflows.

---

## How to Contribute

### 1. Propose Changes
- Open an issue describing the improvement
- Clearly explain the motivation and scope

### 2. Make Changes
- Create a feature branch
- Follow existing file naming conventions
- Keep changes focused and well-documented

### 3. Submit a Pull Request
- Reference the related issue
- Describe what was changed and why
- Include validation steps if applicable

---

## Contribution Areas
- Runbooks
- Monitoring improvements
- Automation guardrails
- Postmortem templates
- Documentation clarity

---

## Standards
- Use clear, concise language
- Prefer safety over speed
- Follow blameless culture principles

---

## Code of Conduct
All contributors are expected to maintain a respectful and professional tone.
